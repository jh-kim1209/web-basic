# 1주차: 자바스크립트의 작동 방식과 실행 환경

## 📖 핵심 요약
자바스크립트 엔진이 소스 코드를 읽어들이고 실행하는 내부적인 메커니즘을 심도 있게 이해하고, 가장 중요한 핵심 개념인 **클로저(Closure)**와 **실행 컨텍스트(Execution Context)**를 학습합니다.

*   **실행 컨텍스트 (Execution Context)**: 자바스크립트 코드가 실행되기 위해 필요한 환경 정보를 모아둔 객체로, 실행 시점에 콜 스택(Call Stack)에 푸시되고 완료되면 팝(Pop)됩니다.
*   **호이스팅 (Hoisting)**: 변수 및 함수 선언이 렉시컬 스코프 최상단으로 끌어올려지는 것처럼 동작하는 현상입니다. `let`과 `const`는 TDZ(Temporal Dead Zone) 제약을 받으므로 선언 전에 접근할 수 없습니다.
*   **클로저 (Closure)**: 외부 함수가 종료된 후에도 외부 함수의 렉시컬 환경(Lexical Environment)을 기억하고 참조할 수 있는 중첩 함수의 상태 또는 기능입니다. 데이터 은닉 및 상태 캡슐화에 사용됩니다.

## 🎯 학습 목표
1.  실행 컨텍스트와 변수 환경(Variable Environment), 렉시컬 환경(Lexical Environment)의 구성을 설명할 수 있다.
2.  호이스팅과 TDZ의 원리를 분석하여 오류가 없는 스크립트를 작성할 수 있다.
3.  클로저를 이용하여 전역 변수 사용을 피하고 안전하게 변수 상태를 은닉하는 법을 구현할 수 있다.

## 🧪 1주차 미션: 캡슐화된 상태 저장소(Store) 모듈 구현
*   **미션 파일**: `mission.js`
*   **요구 사항**: 
    1.  `createStore` 함수는 초기 상태(`initialState` 객체)를 입력받아 새로운 스토어 인스턴스를 반환합니다.
    2.  스토어의 실제 상태(`state`)는 오직 클로저의 내부 변수로만 관리되어야 하며, 외부에서 직접적으로 재할당하거나 변경할 수 없습니다.
    3.  상태 값을 조회할 수 있는 `getState()` 메서드를 제공해야 합니다.
    4.  상태 값을 안전하게 수정(불변성을 준수하며 얕은 복사)하고, 변경 사항이 감지되었을 때 미리 등록된 구독자(Subscriber) 함수들에게 상태 업데이트를 전송하는 `setState(newState)` 메서드를 제공해야 합니다.
    5.  상태 변경을 감지하고 콜백을 등록할 수 있는 `subscribe(listener)` 메서드를 제공해야 하며, 구독을 해제할 수 있는 `unsubscribe` 함수를 반환해야 합니다.

## 💡 학습 및 구현 참고 가이드
*   **상태 은닉화(Encapsulation)**: 내부 변수 `state`와 `listeners`를 클로저 외부로 유출하지 않고 메서드로만 접근할 수 있게 설계합니다.
*   **불변성(Immutability)**: `getState()` 반환 시 및 `setState()`의 상태 병합 시 객체의 얕은 복사(`{ ...state }`)를 활용하여 예기치 못한 상태 변질을 방지합니다.
*   **성능 최적화**: 상태 값에 실제 변화가 있을 때만 `listeners`가 알림을 받아 실행되도록 `setState` 내에서 조건식(`hasChanged`)을 올바르게 감싸서 리스너를 호출해야 합니다.
