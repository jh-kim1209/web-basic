# 2주차 미션: 선언형 데이터 가공 파이프라인 개념 설명서

이 문서는 2주차 미션의 핵심인 **함수형 프로그래밍(Functional Programming)** 방법론을 자바스크립트 환경에서 구현하기 위한 가이드입니다. 1주차에서 배운 **클로저(Closure)** 개념이 어떻게 고차 함수와 커링(Currying)에 재활용되는지 파악하고, 선언형 프로그래밍의 장점을 이해하는 것을 목표로 합니다.

---

## 1. 명령형(Imperative) vs 선언형(Declarative) 프로그래밍

이해도를 높이기 위한 첫걸음은 두 패러다임의 코드 작성 스타일 차이를 체득하는 것입니다.

*   **명령형 (How - 어떻게 할 것인가)**: 
    *   원하는 결과를 얻기 위해 루프(`for`, `while`)를 돌며 배열의 상태를 직접 수정하거나 조건문을 분기합니다.
    *   제어 흐름이 노출되어 있어 코드가 길어지고 버그가 발생할 확률이 높습니다.
*   **선언형 (What - 무엇을 할 것인가)**:
    *   데이터를 어떻게 가공할 것인지에 대한 정의(함수)들을 조립하여 결과를 냅니다.
    *   상태의 직접적인 수정이 없고(Immutable), 파이프라인을 타고 흐르는 데이터의 흐름이 한눈에 보입니다.

## 2. 함수 합성: `pipe`

함수 합성은 수학에서의 합성함수 $f(g(x))$ 와 같습니다. `g`의 출력이 `f`의 입력으로 들어가는 원리입니다.

### 동작 원리 및 개념
여러 개의 단일 인자 함수 `f1, f2, f3`가 있을 때, `pipe(f1, f2, f3)(x)`는 `f3(f2(f1(x)))` 순서로 함수를 실행합니다. 
(왼쪽에서 오른쪽으로 실행 흐름이 이어집니다.)

### 단계별 힌트
이를 자바스크립트의 `Array.prototype.reduce`를 사용해 깔끔하게 합성할 수 있습니다.
```javascript
export function pipe(...fns) {
  // 1. pipe 함수는 인자로 전달받은 여러 함수(fns)를 기억하는 클로저를 형성합니다.
  // 2. 그리고 입력값(x)을 받는 새로운 함수를 반환합니다.
  return function(initialValue) {
    // 3. reduce를 활용하여 초기값부터 fns 배열의 함수들을 차례로 누적(accumulate) 실행시킵니다.
    return fns.reduce((acc, fn) => fn(acc), initialValue);
  };
}
```

## 3. 커링 (Currying)

커링은 **"여러 개의 인자를 받는 함수를 단일 인자를 받는 함수의 체인으로 만드는 것"**입니다.

### 왜 필요한가요?
1.  **재사용성**: 함수의 인자 중 일부를 고정해 둔 '새로운 특화 함수'를 쉽게 생성할 수 있습니다.
2.  **지연 평가(Lazy Evaluation)**: 필요한 모든 인자가 채워질 때까지 함수의 실행을 미룰 수 있습니다.
3.  **조립 용이성**: `pipe` 함수는 인자를 단 하나만 받는 함수들을 연결해야 작동합니다. 다중 인자 함수(예: `filter(predicate, list)`)를 커링하면 단일 인자 함수(`filter(predicate)`)로 변환하여 `pipe` 안에 조립할 수 있습니다.

### 동작 원리 및 힌트
`fn.length` 속성을 사용하면 함수가 원래 기대하는 매개변수 개수(Arity)를 알 수 있습니다.
*   수집된 인자의 개수가 원래 함수의 기대 개수보다 **크거나 같으면** 원래 함수(`fn`)를 즉시 실행합니다.
*   수집된 인자의 개수가 원래 함수의 기대 개수보다 **작으면** 나머지 인자를 추가로 수집할 수 있는 또 다른 클로저 함수를 반환합니다.

```javascript
export function curry(fn) {
  return function curried(...args) {
    // 1. 지금까지 수집된 인자 개수(args.length)가 함수가 정의한 인자 개수(fn.length) 이상인지 비교합니다.
    if (args.length >= fn.length) {
      return fn(...args);
    }
    // 2. 인자가 더 필요하다면, 다음 인자들(nextArgs)을 받아 기존 인자들과 합치는 재귀 클로저 함수를 반환합니다.
    return (...nextArgs) => curried(...args, ...nextArgs);
  };
}
```

## 4. 데이터 원본 격리 (Immutability, 불변성)

함수형 프로그래밍에서 순수 함수는 **부작용(Side Effect)이 없어야 하며, 인자로 받은 데이터를 직접 조작(Mutate)해서는 안 됩니다.**

자바스크립트의 표준 배열 메서드 중 일부는 원본 배열을 변형시킵니다.
*   `filter`와 `map`: 항상 새로운 배열을 반환하므로 안전합니다.
*   `sort`: **원본 배열의 순서를 직접 바꿉니다.** (부작용 발생!)

### ⚠️ 위험한 예시
```javascript
// 원본 배열 list가 직접 정렬되어서 원본 데이터가 훼손됩니다!
export const sort = curry((compareFn, list) => {
  return list.sort(compareFn); 
});
```

### ✅ 안전한 예시 (불변성 보장)
```javascript
// 전개 연산자([...list]) 또는 slice()를 활용해 복사본을 만든 뒤 정렬을 수행합니다.
export const sort = curry((compareFn, list) => {
  return [...list].sort(compareFn);
});
```

## 💡 이해도와 숙련도를 극대화하는 자율 도전 과제

1주차를 완료하고 2주차로 접어들면서, 단순 미션 통과를 넘어 깊은 실력을 기르기 위한 가이드라인입니다.

### 1단계: 직접 바닐라 코드로 재구현하기
*   `Array.prototype.map`과 `Array.prototype.filter` 같은 내장 메서드를 사용하지 말고, **`reduce`만 사용해서 `map`과 `filter` 도우미 함수를 직접 구현**해 보세요.
*   예시: `reduce((acc, cur) => ..., [])`
*   이를 통해 배열 고차 함수의 진정한 근간이 되는 `reduce`에 대한 지배력을 키울 수 있습니다.

### 2단계: 파이프라인 디버거 만들기
*   `pipe`를 통해 함수를 여러 개 엮을 때, 중간에 어떤 데이터가 흐르고 있는지 로깅하고 싶을 때가 있습니다.
*   파이프라인 중간에 끼워 넣을 수 있는 `tap` 또는 `log` 함수를 만들어 보세요.
```javascript
const tap = curry((tag, x) => {
  console.log(`[${tag}]:`, x);
  return x; // 파이프가 끊기지 않도록 받은 값을 그대로 반환
});

// 사용 예시
pipe(
  filter(user => user.age >= 20),
  tap('Filter 완료'),
  map(user => ({ name: user.name })),
  tap('Map 완료')
)(users);
```

### 3단계: 커링의 엣지 케이스 처리
*   만약 원래 함수의 매개변수 중 기본값(Default Parameter)이 지정되어 있거나 가변 인자(`...args`)를 쓰는 경우, `fn.length`를 통한 커링은 어떻게 작동할까요?
*   인자가 아예 전달되지 않고 빈 괄호 `()`로 계속 호출될 때의 대처 방법을 고민하고 테스트해 보세요.
