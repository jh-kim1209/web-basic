# Milestone 1: 데이터 파이프라인 통합 구축 (1~2주차)

## 📌 목표
1주차에 작성한 클로저 기반의 **상태 저장소(Store)**와 2주차에 작성한 **함수 합성 및 커링(pipe, curry) 데이터 파이프라인**을 결합하여, 유저 정보가 변경될 때마다 정제된 데이터를 자동으로 가공해 출력하는 견고한 데이터 파이프라인 모듈을 완성합니다.

---

## 🛠️ 요구 사항
1.  **스토어 연동**: `createStore`를 호출하여 사용자 초기 데이터 배열을 저장하는 `userStore` 인스턴스를 빌드합니다.
2.  **구독 설정**: `userStore`가 변경될 때마다 동작할 리스너 콜백을 `subscribe` 메서드로 부착합니다.
3.  **데이터 정제 파이프라인 결합**: 스토어 상태 변경 리스너가 호출될 때마다, 2주차의 `processUsers(users)` 파이프라인 함수를 실행하여 가공(20세 이상 필터링 -> 이름/이메일만 매핑 -> 알파벳 정렬) 완료된 결과물을 반환하고 화면이나 터미널에 로깅하는 함수를 완성하십시오.
4.  **불변성 보장**: 이 모든 결합 과정에서 원본 `users` 배열 객체의 불변성을 유지해야 하며, 전역 상태의 무분별한 훼손이 발생하지 않는지 스스로 검토해야 합니다.
