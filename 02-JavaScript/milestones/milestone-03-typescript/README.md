# Milestone 3: TypeScript 리팩토링 및 Jest 테스트 보완 (5~6주차)

## 📌 목표
앞서 구축한 모든 바닐라 자바스크립트 스토어, 함수형 데이터 파이프라인, 그리고 이벤트 리스너 트래커 클래스 등을 **TypeScript** 정적 타입 파일(`.ts`)로 전면 전환하고, **Jest** 단위 테스트(Unit Test) 세트를 작성하여 테스트 커버리지를 끌어올림으로써 엔지니어링 완성도를 고도화합니다.

---

## 🛠️ 요구 사항
1.  **타입 정의 및 정적 검사**:
    *   사용자 데이터(`User`), 정렬 기준(`SortCriteria`), 리스너 등록 상태(`ListenerRegistry`) 등의 엄격한 인터페이스(Interface) 및 타입 별칭(Type Alias)을 수립합니다.
    *   tsconfig.json의 컴파일 룰을 만족하여 타입 경고(`any` 사용 최소화)가 전혀 발생하지 않도록 컴파일러 경고를 모두 해결합니다.
2.  **단위 테스트 커버리지**:
    *   스토어 상태 변경, 데이터 정제 필터 함수, 이벤트 등록/해제 관리 객체(`SafeRegistry`)의 단위 테스트 스크립트를 작성합니다.
    *   경계값 예외 처리(빈 배열, 널값(Nullish), 비정상 데이터 입력 등) 상황에 대한 엣지 케이스 테스트 케이스를 다각도로 설계합니다.
3.  **코드 린트 및 정렬**:
    *   ESLint 및 Prettier 실행 스크립트(`npm run lint`, `npm run format`)를 가동하여 공통 정적 규칙에 따라 최종 코드를 조율합니다.
