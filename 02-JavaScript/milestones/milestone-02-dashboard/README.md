# Milestone 2: Web Workers 기반 실시간 대시보드 구축 (3~4주차)

## 📌 목표
3주차의 표준 **웹 컴포넌트(Web Components)** 테이블 객체와 4주차의 **Web Workers** 멀티스레드 비동기 통신을 융합하여, 대량의 데이터 처리를 UI 렌더링 스레드 부하 없이 안전하게 수행하는 실시간 대시보드 컴포넌트를 설계합니다.

---

## 🛠️ 요구 사항
1.  **사용자 정의 컴포넌트 확장**: `<user-dashboard>` Custom Element를 빌드하고 Shadow DOM을 부착합니다.
2.  **데이터 조회 및 비동기 처리**: `fetch` API를 사용하여 목업 사용자 API로부터 대량의 데이터 세트를 비동기로 가져옵니다.
3.  **워커 스레드 위임**: 사용자가 대시보드 상단에 검색 키워드를 입력할 때마다, 입력 핸들러가 해당 데이터 세트를 **Web Worker**로 즉각 전달하고, 연산(문자열 포함 검사 및 정렬)은 메인 스레드 밖(백그라운드)에서 수행하도록 구성합니다.
4.  **UI 동적 렌더링**: 워커로부터 `onmessage` 결과가 도착하면, 그 결과 데이터를 3주차에 개발한 `<user-table>`의 `users` 속성에 주입하여 화면을 동적으로 리프레시합니다.
5.  **성능 대조**: 대용량 검색 필터링을 동기적으로 수행할 때와 Web Worker를 통해 비동기로 처리할 때 화면 버벅임 및 FPS 저하 차이를 분석하고 체득합니다.
